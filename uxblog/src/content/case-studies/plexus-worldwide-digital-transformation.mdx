## Three redesigns, each one smarter than the last.

The first redesign set the baseline. The second incorporated A/B test data and ambassador feedback. The third implemented the full design system with Angular component parity. None of them started from scratch — each one was strategic evolution built on what we learned from real users and real data.

## A design system that improved handoff efficiency by 60%.

The atomic design system wasn't just a Sketch library — it was a shared language. Atoms became Angular components. Design tokens became SCSS variables. The result was a mapping so direct that developers could implement designs without re-interpreting them. That's the kind of efficiency that compounds across every future project.

</div>

## An ambassador site builder that empowered thousands of non-technical users.

Every Plexus ambassador needed a personal website. I designed a replicated site system with controlled personalization — templates with real customization, brand guardrails built in, visual preview, and SEO optimization. Non-technical users could build a professional web presence without touching code or calling support.

</div>

## A ZenDesk integration suggestion that cut the customer service team by 60%.

I noticed that the most common support contacts were answerable through better self-service design. I proposed a custom-themed ZenDesk integration with structured knowledge base architecture. The company implemented it — and reduced their customer service headcount by 60%. Design thinking applied to operations isn't overhead. It's leverage.
